# Dispatch на iPhone + синхронизация — подробная инструкция

Что получится в итоге: приложение открывается на iPhone как иконка на экране «Домой»,
а данные автоматически синхронизируются между Mac и iPhone через интернет.

Понадобится 2 бесплатных аккаунта: **GitHub** (хостинг) и **Supabase** (синхронизация).
Время: ~20–30 минут. Делается один раз.

Порядок такой, и его важно соблюсти:
1. Разместить приложение в интернете (GitHub Pages).
2. Перенести твои текущие данные в размещённое приложение.
3. Настроить облако (Supabase).
4. Включить синхронизацию на Mac, потом на iPhone.

---

## ⚠️ Важно про данные (прочитай до начала)

Данные приложения хранятся в браузере и **привязаны к адресу страницы**. Сейчас ты
открываешь приложение как файл на Mac — у этого «адреса» свои данные. Когда откроешь
приложение по новому адресу `github.io`, там сначала будет **пусто**. Поэтому в шаге 2
мы перенесём данные через файл-бэкап. Ничего не потеряется, если идти по порядку.

---

## ЧАСТЬ 1. Разместить приложение в интернете (GitHub Pages)

### 1.1. Аккаунт GitHub
- Открой **github.com**. Если аккаунта нет — **Sign up**, регистрация по почте, бесплатно.

### 1.2. Создать репозиторий (это «папка проекта» в интернете)
1. Вверху справа нажми **+** → **New repository**.
2. **Repository name**: `dispatch` (или любое).
3. Тип: оставь **Public** (публичный). *(Секретов в файлах нет — ключи вводятся в самом приложении, не в коде.)*
4. Нажми **Create repository**.

### 1.3. Загрузить файлы приложения
Нужно загрузить **всё содержимое папки проекта** с сохранением структуры папок
(`index.html`, папки `css`, `js`, `assets`, файл `manifest.json` и остальные).

1. На странице репозитория нажми ссылку **uploading an existing file**
   (или **Add file → Upload files**).
2. Открой на Mac папку проекта (`/Users/debilus/claude projects/dispatch`).
3. Выдели всё внутри неё (⌘A) и **перетащи** в окно браузера.
   - Современный GitHub принимает и папки целиком, структура сохранится.
   - Если папки не перетаскиваются — перетащи сначала `index.html` и `manifest.json`,
     затем по очереди папки `css`, `js`, `assets` (можно зайти в каждую и перетащить её файлы,
     указав путь — но проще через **GitHub Desktop**, см. ниже).
4. Внизу нажми зелёную **Commit changes**.

> 💡 Проще, без возни с папками: установи **GitHub Desktop** (desktop.github.com) → «Add local
> repository» → выбери папку проекта → Publish. Он загрузит всё со структурой автоматически.

> 💡 Ещё проще, если GitHub кажется сложным: **Netlify Drop** — открой `app.netlify.com/drop`,
> перетащи туда **всю папку проекта**, и сразу получишь рабочий адрес (бесплатно, аккаунт
> можно создать потом). Тогда часть про Pages ниже можно пропустить — адрес даст Netlify.

### 1.4. Включить GitHub Pages
1. В репозитории открой вкладку **Settings** (вверху).
2. Слева — **Pages**.
3. **Source** → **Deploy from a branch**.
4. **Branch** → выбери `main`, папка `/ (root)` → **Save**.
5. Подожди ~1 минуту, обнови страницу — вверху появится:
   **«Your site is live at `https://ТВОЙ-ЛОГИН.github.io/dispatch/`»**.
   Это и есть адрес приложения. Скопируй его.

### 1.5. Проверить
- Открой этот адрес в браузере на Mac — должно открыться приложение (пустое).

## ЧАСТЬ 2. Перенести твои текущие данные

1. Открой приложение там, **где сейчас твои данные** (твой обычный файл на Mac).
2. Вкладка **Клиенты** → блок **«Бэкап данных»** → **⬇ Экспорт в файл**.
   Скачается файл `dispatch-backup-….json`.
3. Открой **новый адрес** `https://…github.io/dispatch/` (из шага 1.4).
4. Там: **Клиенты → Бэкап данных → ⬆ Импорт из файла** → выбери скачанный JSON.
5. Приложение перезагрузится — данные на месте. Теперь твои данные «живут» по новому адресу.

*(Если ты уже пользуешься приложением именно по будущему адресу — этот шаг пропусти.)*

## ЧАСТЬ 3. Настроить облако (Supabase)

### 3.1. Аккаунт и проект
1. Открой **supabase.com** → **Start your project** → войди через GitHub или почту (бесплатно).
2. **New project**:
   - **Name**: `dispatch`.
   - **Database Password**: придумай и сохрани (понадобится только Supabase, нам — нет).
   - **Region**: ближайший к тебе.
   - **Create new project**. Подожди 1–2 минуты, пока проект поднимется.

### 3.2. Создать таблицу (один раз)
1. Слева в меню — иконка **SQL Editor**.
2. **New query**, вставь это целиком и нажми **Run** (или ⌘↵):

```sql
create table if not exists public.dispatch_sync (
  id          text primary key,
  data        jsonb,
  device      text,
  updated_at  timestamptz default now()
);

alter table public.dispatch_sync enable row level security;

create policy "dispatch anon access"
  on public.dispatch_sync for all
  to anon using (true) with check (true);

-- мгновенная синхронизация (realtime):
alter publication supabase_realtime add table public.dispatch_sync;
```

Должно написать **Success**. *(Если последняя строка про `supabase_realtime` ругается —
не страшно: синк всё равно работает через опрос раз в 20 секунд. Можешь удалить эту строку
и выполнить остальное.)*

### 3.3. Взять адрес и ключ
1. Слева внизу — **Project Settings** (шестерёнка) → **API** *(или раздел «API Keys / Data API»)*.
2. Скопируй два значения:
   - **Project URL** — вида `https://abcdxyz.supabase.co`.
     **⚠️ Только базовый адрес, без `/rest/v1/` в конце!** Если скопировал из примеров API
     что-то вроде `https://abcdxyz.supabase.co/rest/v1/` — убери хвост `/rest/v1/`.
     *(Приложение теперь и само отрезает лишнее, но лучше вводить чисто.)*
   - **anon** **public** ключ — длинная строка, обычно начинается на `eyJ…`.
     *(Если видишь только новые ключи — подойдёт ключ с пометкой `publishable`/`anon`.
     НЕ бери `service_role` — он секретный.)*

## ЧАСТЬ 4. Включить синхронизацию

### 4.1. На Mac (где уже есть данные после Части 2)
1. Открой приложение по адресу `…github.io/dispatch/`.
2. **Клиенты** → блок **«Синхронизация (облако)»**.
3. Заполни:
   - **Project URL** — из шага 3.3;
   - **anon public key** — из шага 3.3;
   - **Код синхронизации** — придумай секретную строку, например `dispatch-pavel-7x9q`.
     **Запомни её — она должна быть ОДИНАКОВОЙ на всех устройствах.**
4. Нажми **Подключить**.
5. Появится вопрос (облако ещё пустое или есть данные):
   - выбери **выгрузить локальные в облако** (ОК/Отмена — см. текст в окне; нам нужно
     отправить данные Mac в облако).
6. Статус должен показать **«✓ синхронизировано»**.

### 4.2. На iPhone
1. В Safari открой тот же адрес `…github.io/dispatch/`.
2. Кнопка **«Поделиться»** (квадрат со стрелкой) → **На экран «Домой»** → Добавить.
   Появится иконка-приложение; открой его с экрана «Домой» (так оно полноэкранное).
3. **Клиенты** → **«Синхронизация (облако)»** → введи **тот же** URL, ключ и **тот же код**.
4. **Подключить** → при вопросе выбери **загрузить из облака**.
5. Приложение перезагрузится — данные с Mac появятся на iPhone.

### 4.3. Дальше — автоматически
- Любое изменение на одном устройстве в течение 1–2 секунд уходит в облако.
- На другом устройстве оно подтягивается почти сразу (realtime) или максимум за ~20 секунд (опрос).
- Кнопки **⬆ Выгрузить / ⬇ Загрузить** — ручная синхронизация в любой момент.

## Обновление приложения в будущем
Если файлы приложения поменялись — загрузи изменённые файлы в тот же репозиторий
(**Add file → Upload files** → Commit), GitHub Pages обновится за пару минут.
На устройствах просто переоткрой приложение.

## Если что-то не так
- **Рядом с кнопками «⚠ …»** — это текст ошибки от Supabase. Частые причины: опечатка в URL
  или ключе; не выполнен SQL из 3.2; взят `service_role` вместо `anon`.
- **Не подтягивается с другого устройства** — проверь, что **код синхронизации одинаковый**.
  Если realtime не включился — подожди до 20 сек (работает опрос) или нажми **⬇ Загрузить**.
- **На iPhone «пусто»** — убедись, что на Mac статус был «✓ синхронизировано», а на iPhone
  при подключении выбрал **загрузить из облака**.
- **Данные пропали после смены адреса** — это нормально для нового адреса; вернись к Части 2
  и импортируй бэкап. Старые данные никуда не делись (они под старым адресом).

## Безопасность
- URL и ключ нигде в коде/репозитории не лежат — ты вводишь их в приложении, они хранятся
  только на твоём устройстве.
- Доступ к данным защищён **кодом синхронизации** (это `id` строки в таблице). Держи в секрете
  и ключ, и код.
